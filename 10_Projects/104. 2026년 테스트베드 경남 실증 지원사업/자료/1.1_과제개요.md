### 과제 개요

#### ① 적용대상/현장

○ 본 과제는 금속가공·정밀가공 공정에서 운용 중인 MCT(Machining Center) 1대를 대상으로, 현장 운영자·설비관리자·교육담당자가 실시간 모니터링·동작 재현·교육/훈련을 수행할 수 있도록 Unity 기반 디지털트윈을 구축·실증하는 사업임

- 적용 장비: MCT 1대(대상 기종·Controller 모델은 협약 시 확정)
- 실증 장소: 경상남도 창원시 소재 제조 현장(한국폴리텍대학 실습장 또는 협력기업 가공라인)
- 운영 주체: 주관기관(㈜알파에듀)이 시스템 구축·운영을 담당하고, 공동수행기관(한국폴리텍대학)이 현장 데이터 접근·수집 환경 제공 및 실증 운영을 지원함
- 운영 조건: 기존 모니터링 시스템과 병행운영(무중단) 원칙을 적용하며, 현장 운영자(운전·시나리오 수행)와 설비관리자(승인·협의)를 지정하여 8개월간 단계적으로 수행함
- 네트워크 구성: Controller ↔ Edge PC(데이터 수집) ↔ 미들웨어(정규화·스트리밍) ↔ Unity(3D 재현) 구조로 구성하되, 현장 보안정책(망 분리 등)을 준수함

#### ② 입력데이터

○ 입력데이터는 MCT Controller에서 출력되는 파라미터·상태·알람 정보이며, 대표 항목(10종)은 다음과 같음

대표 파라미터(10종)	설명	Unity 매핑 예시
주축 회전수(RPM)	현재 스핀들 회전속도	주축 회전 애니메이션 속도에 1:1 동기화
이송속도(Feed)	축 이동 시 테이블/공구 이송률	테이블·축 이동속도 반영 및 UI 트렌드 게이지 표시
주축부하(Spindle Load)	주축 모터 토크 대비 현재 부하율(%)	부하 게이지 색상 변환(정상→주의→경보 임계치 기반)
서보부하(Servo Load)	각 축(X/Y/Z) 서보모터 부하율	축별 부하 상태 바/색상 표시
축 위치(X/Y/Z)	각 축의 현재 좌표값(기계좌표/워크좌표)	3D 모델 내 축·테이블 위치 실시간 이동 반영
프로그램 번호/블록	현재 실행 중인 NC 프로그램 및 블록 번호	UI 상단 정보 패널에 실행 프로그램 표시
운전상태(Mode)	Run / Idle / Alarm 등 장비 동작 모드	상태머신 전이에 따른 3D 모델 동작·정지·경보 연동
알람 코드/메시지	발생 중인 알람 번호 및 상세 메시지	팝업 경보 + 이력 로그 + 3D 해당 부위 하이라이트
오버라이드(Feed/Spindle)	운전자 수동 조정 비율(%)	게이지/슬라이더 UI에 실시간 반영
가공시간/사이클 상태	현재 사이클 경과시간, 사이클 완료/진행 여부	사이클 타이머 표시 및 완료 시 상태 전이 트리거

- 데이터 수집 주기: 100ms~500ms 단위(파라미터별 상이, 데이터사전에서 정의)
- 데이터 포맷: Controller 프로토콜(Fanuc FOCAS / Siemens OPC UA 등)로 수집 후 미들웨어에서 JSON/경량 포맷으로 정규화하여 Unity에 스트리밍함
- 알람·이벤트: 상태 전이(Idle→Run→Alarm 등) 발생 시 이벤트 기반으로 즉시 전송하며, 이력 로그에 타임스탬프와 함께 기록함

#### ③ 핵심기능

○ 본 과제의 디지털트윈은 단순 선언이 아닌, 아래 4개 범위로 구체화하여 구현함

##### 실시간 상태동기화(필수)
- Controller → Collector(Edge) → Middleware(정규화·버퍼·재전송) → Unity(3D 반영)의 end-to-end 데이터 파이프라인을 구축함
- 목표 성능: End-to-end 지연시간 P95 0.5초 이내, 데이터 누락률 0.1% 이하
- 미들웨어는 시퀀스 ID 기반 누락 탐지, 버퍼링/재전송, keep-alive 기반 자동 재연결(10초 이내 복구), 타임스탬프 동기화 기능을 포함함

##### 이벤트·알람 가시화(필수)
- 알람 발생/해제, 상태 전이(Idle/Run/Alarm)를 Unity 3D 화면 및 UI 패널에 실시간 표시함
- 알람 이력 로그(발생시각·코드·메시지·해제시각)를 저장하고 조회 기능을 제공함
- 상태 전이 일치율 95% 이상(Controller 실제 상태 대비 Unity 표시 정확도)을 달성 목표로 설정함

##### 가공경로 재현(선택)
- 축 위치(X/Y/Z) 및 프로그램 블록 데이터 접근이 가능한 경우, Unity 3D 공간에서 공구 이동 궤적을 시각적으로 재현함
- Controller 데이터 접근 범위에 따라 적용 여부를 결정하며, 데이터사전 확정 단계(3~4월)에서 범위를 확정함

##### 분석·예지(후속 로드맵)
- 본 과제 실증 범위에서는 실시간 모니터링·동기화·가시화에 집중하되, 수집된 파라미터 데이터를 로그로 축적함
- 실증 종료 후 축적 데이터를 기반으로 예지보전(PdM)·이상징후 탐지 등 분석 고도화를 후속 사업화 로드맵으로 연결함

#### ⑤ 실증 범위

○ 본 과제의 실증 범위는 공통목표 **'테스트베드 현장적용 1건 완료'**를 최종 목표로 설정하며, 8개월(2026.3~2026.10) 내 아래 5단계를 순차 수행함

단계	기간	주요 활동	완료 기준
설치	3~4월	Edge PC·미들웨어·Unity 환경 구축, 네트워크 연결, 보안 협의	설치 완료 확인서, 네트워크 구성도
연동	4~6월	Controller 데이터 수집 확인, 데이터사전 확정, Unity 3D 모델·상태머신 연동	데이터사전 v1.0, 연동 체크리스트
시운전	6~7월	병행운영 적용, 기본 시나리오(정상 운전·상태 전이) 검증	시운전 체크리스트, 현장 사진
실증	7~9월	정량 KPI 측정(지연·누락·복구·일치율·안정성 등), 장애주입 시험, 8시간 연속 운전	KPI 측정 로그, 시나리오 결과표
최종검증	10월	시험성적서 작성, 성능 튜닝/UI 보완, 최종 결과보고서 제출	시험성적서, 결과보고서, 매뉴얼

- 실증 종료 시점에 사업화율 130% 이상(목표 매출 1.56억원), 지식재산권 1건(미들웨어 품질 로직/매핑 검증 방법 출원), 신규채용 2명, 순고용증가 2명, 컨설팅 1건, 협업 네트워크 1건, 경남 디지털위크 참가 1건 등 공통목표 전항목 달성을 전제로 사업화 단계까지 연결함
- 도입의사 확약서를 징구한 실증 적용처에서 수요기업과의 실증 협약 및 현장적용 완료를 확인하며, 현장적용 사진 및 실증 데이터를 포함한 결과보고서를 최종 증빙으로 제출함
