- # 5. 사업화 전략
자료와 아래 작성 지침에 근거하여 5. 사업화 전략을 작성해줘. 작성 지침을 전체적으로 확인한 후 작성해줘. 

(작성 지침)

- 구체적인 수치와 일정 제시할 것
    
- 실현 가능한 현실적인 계획을 수립할 것
    
- 수요확약서 기준으로 협의된 기업명 기재할 것
    


- 사업화는 “무엇을(패키지) / 누구에게(고객군) / 얼마에(단가) / 어떻게 팔지(채널) / 유지보수(연간)” 5요소가 모두 들어가야 한다.
    
- 수요확약서가 있다면 “기업명 + 도입 목적(운영/교육) + 도입 시기 + 예상 대수”를 본문에 직접 반영한다.
    
- 수치 산정은 ‘희망’이 아니라 **가정(도입률, 단가, 유지보수율)**을 명시해 현실성을 확보한다.

## 5.1. 생산 계획

### 5.1.1. 2027년

… (표/수치 항목 유지)

[(작성지침)]

- 시장점유율은 “목표시장(SAM) 대비”로 산정하고, SAM 정의를 앞 장(시장규모)와 일치시킨다.
    
- 판매량(대)은 “장비 1대당 1라이선스/1패키지” 등 **판매 단위 정의**를 먼저 적는다.
    
- 판매단가는 “초기 구축비 + 연간 유지보수”로 분리해 제시하면 신뢰도가 높다.
    
- 해외는 국가/권역을 1~2개로 제한해 현실적으로 제시한다(예: 일본/동남아 등).
    

### 5.1.2. 2028년 / 5.1.3. 2029년

[(작성지침)]

- 연도별 성장 근거를 반드시 붙인다: “레퍼런스 확보→다장비 확장→파트너 채널 확대” 같은 단계 논리.
    
- 판매량 증가가 인력/납기/지원능력과 충돌하지 않도록, 4.5 인력계획과 연결한다.

## 5.2. 투자 계획

(작성 지침)

- 전체적인 상황을 종합적으로 검토하여 실질적으로 작성할 것
    

[(작성지침)]

- 투자계획은 “고정비(개발/인력) + 변동비(현장 설치/출장/장비) + CAPEX(서버/워크스테이션)”로 분리한다.
    
- 매출원가는 SW 과제 특성상 재료비보다 “노무비/외주비/현장비” 비중이 높음을 전제로 항목을 현실적으로 구성한다.
    
- 연도별로 “매출 성장에 따른 인력/인프라 확충” 논리가 보이게 작성한다.
    

### 5.2.1~5.2.3(연도별)

[(작성지침)]

- 판매관리비에 “영업/마케팅/파트너 교육/고객지원”을 반영한다.
    
- CAPEX는 과도하게 쓰기보다 “필수 장비(고성능 PC, 테스트 장비)” 위주로 합리화한다.

## 5.3. 사업화 전략

(작성 지침)

- 전체적인 과제 내용을 종합적으로 판단하여 구체적으로 작성할 것
    

[(작성지침)]

- 전략은 **3단 로드맵**으로 작성한다:
    
    1. 1호 실증 레퍼런스 확보(성과지표 공개 가능 범위 포함)
        
    2. 기종 확장 템플릿화(Controller/태그/3D)
        
    3. 다장비/라인 확장 + 유지보수/구독 모델
        
- 가격 전략은 “기본 패키지(단일장비) / 확장(장비추가) / 유지보수(연간)” 3단으로 구성한다.
    
- 경쟁우위는 “추가 센서 최소화, Unity 기반 교육 확장, 현장형 미들웨어 안정성” 3개로 고정해 반복 강조한다.

[(작성지침)]

- 상용화 형태는 ①프로젝트형(구축) ②라이선스형(연간) ③구독형(SaaS) 중 1~2개를 선택해 명확히 한다.

### 5.3.2. 상용화 능력 및 자원보유

[(작성지침)]

- 자원은 “인력(직무별) / 기술자산(모듈·템플릿) / 협력망(폴리텍·현장기업) / 운영체계(QA·지원)” 4개로 나눠 쓴다.
    
- 과제 산출물이 바로 상용화 자산으로 전환되는 연결고리(템플릿화, 매뉴얼화, 설치 자동화)를 명시한다.
    

### 5.3.3. 상용화 계획 및 일정

[(작성지침)]

- 일정은 “1호 고객 납품 → 2~3호 확산 → 파트너 채널 → 해외/권역 확장” 순으로 단계화한다.
    
- 각 단계마다 “목표 고객 수(또는 장비 대수) + 매출 목표 + 준비 항목(기술/영업/지원)”을 3줄로 요약한다.
