## 1.1. 과제 개요

첨부 자료와 아래의 작성 지침에 의거하여 1.1 과제 개요를 작성해줘


## 작성지침

- **반드시 5개 소제목**으로 고정해 작성한다. (예: ①대상/적용현장 ②입력데이터 ③핵심기능 ④산출물/결과 ⑤적용 시나리오)
    
- **용도/적용분야**는 “어떤 공정(금속가공/정밀가공 등) + 어떤 장비(MCT) + 어떤 사용자가(운영자/관리자/교육담당) + 무엇을 하려는지(실시간 모니터링·동작 재현·교육)”을 한 문장에 넣는다.
    
- “디지털트윈”을 선언만 하지 말고 **Twin 범위**를 명시한다:
    
    - _실시간 상태동기화(필수)_ / _이벤트·알람 가시화(필수)_ / _가공경로 재현(선택)_ / _분석·예지(후속)_
        
- “Controller 파라미터”는 **대표 8~12개**를 예시로 적고, 각 파라미터가 Unity에서 무엇을 움직이는지(매핑)를 2~3개만이라도 넣는다.
    
- “실증 사업”답게 **실증 장소/대상 기종/운영 조건(운전시간, 네트워크 구성)**을 한 단락으로 요약한다.

5개 세부항목으로 고정한다(예시 틀 그대로 사용 권장):

1. **적용대상/현장**(MCT 1대, 실증 장소/운영자)
    
2. **입력데이터**(Controller 파라미터·알람·상태)
    
3. **핵심기능**(실시간 수집–전송–Unity 동기화–3D 재현–모니터링)
    
4. **산출물**(3D 모델, Unity Twin, 데이터 연동 모듈, 시험성적서/결과보고서)
    
5. **실증 범위**(현장적용 1건 완료를 목표로 운영/검증) ← 공통목표 “테스트베드 현장적용 1건”을 여기에서 미리 박아넣는다.

## 1.2. 과제의 중요성과 파급효과

첨부 자료와 아래의 작성 지침에 의거하여 과제의 중요성과 파급효과를 작성해줘


### 1.2.1. 과제의 중요성

(작성 지침)

- 왜 이 과제가 필요한지, 해결하고자 하는 문제가 무엇인지 명확히 제시
    
- 5개 세부 항목 작성할 것
    

[(작성지침)]

- **문제정의 → 원인 → 현장 피해(손실) → 해결전략 → 기대효과** 순서로 쓴다.
    
- **5개 세부 항목**은 아래 중에서 고정 선택 권장(심사 관점에서 강함):
    
    1. 현장 운영 문제(2D 수치 모니터링 한계, 직관성 부족)
        
    2. 데이터 활용 문제(Controller 데이터는 있으나 통합/표준/가시화 부족)
        
    3. 교육/훈련 문제(장비 실습 한계, 안전/비용/가동중단 이슈)
        
    4. 확장/확산 문제(기종별 폐쇄성, 유지보수/확장 비용)
        
    5. 지역 산업 문제(경남 주력산업의 공정/장비 데이터 활용 수준 제고 필요)
        
- 각 항목은 **“현재 상태(정성/정량) + 발생 리스크(품질/정지/안전/인력) + 과제에서 해결하는 포인트(한 줄)”**로 구성한다.
    
- 가능하면 항목 중 최소 2개는 **수치/지표로** 표현한다(예: 알람 대응시간, 교육시간, 장비 중단 비용, 데이터 지연 등). 수치가 없으면 **지표 정의(어떻게 측정할지)**를 대신 제시한다.

### 1.2.2. 과제의 파급효과

#### 1.2.2.1. 기술적 측면

(작성 지침)

- 해당 기술의 향상, 다른 기술에의 파급효과 등
    

[(작성지침)]

- “기술향상”은 막연히 쓰지 말고 **기술요소별 파급**으로 쪼갠다(최소 4개):
    
    - 데이터 수집/정규화(태그 표준화, 단위 통일)
        
    - 스트리밍/동기화(지연·누락·재연결)
        
    - 3D 실시간 렌더링/상태머신(Unity 구현)
        
    - 이벤트/알람/로그 체계(추적성)
        
- 각 요소마다 **현재 한계 → 과제에서 달성할 개선 → 확장 가능 분야**를 3줄 이내로 작성한다.
    
- 타 기술 파급은 “예지보전/이상탐지/에너지관리/OEE/MES 연동”처럼 **후속 기술 로드맵**과 연결한다.
    

#### 1.2.2.2. 경제적․산업적 측면

(작성 지침)

- 시장 창출 및 일자리 창출 효과, 비용 절감 등 경제적 효과, 산업 발전 영향
    

[(작성지침)]

- **경제효과는 최소 3가지**로 분리해 작성한다: ①비용 절감 ②매출 창출 ③고용/인력양성
    
- 비용절감은 “다운타임 감소/알람 대응시간 감소/교육 비용 감소/시운전 시간 감소” 중 2개 이상을 선택하고, **절감 구조(왜 줄어드는지)**를 설명한다.
    
- 매출 창출은 **패키지 구성(3D 템플릿+연동 모듈+Unity 뷰어+가이드)**을 제시하고, “단일 장비→다장비→라인 확장” 단계별로 **단가/유지보수 형태(구독/연간 유지보수)**를 간단히 언급한다.
    
- 일자리/인력양성은 “3D 모델러/Unity 개발/현장 데이터 엔지니어” 등 **직무 단위**로 쓰고, 지역 기반(경남) 인력양성과 연결한다.
    

#### 1.2.2.3. 디지털 혁신거점(디지털 산업) 측면

(작성 지침)

- 경남 주력산업(제조, 자동차, 조선, 우주항공)에 대한 기여도
    
- 디지털 혁신거점 과제로서의 역할 및 필요성
    
- 디지털 산업 생태계에 미치는 파급효과
    
- 정책 및 정부 지원과 연계성
    

[(작성지침)]

- 경남 주력산업별로 **“MCT 사용 공정 예시 + 적용 가치(운영/품질/교육)”**를 1~2줄씩 반드시 쓴다.
    
    - 자동차: 부품 가공 라인, 공정조건 변화 대응
        
    - 조선: 대형 부품/치공구 가공, 숙련 이전
        
    - 우주항공: 정밀가공/품질 추적성, 공정 신뢰성
        
    - 일반 제조: 다품종 소량, 셋업/전환 효율
        
- 혁신거점 과제 역할은 “실증 레퍼런스 + 표준/데이터사전 + 교육 콘텐츠” 3종을 **거점 자산**으로 정의한다.
    
- 생태계 파급은 “지역 SI/장비업체/교육기관/수요기업”을 나열하고, **각 주체가 얻는 이익**을 1줄씩 제시한다.
    
- 정책 연계는 “스마트제조, 제조DX, 인력양성, 데이터 표준화” 키워드로 **사업 목표와 직접 연결되는 문장**으로 마무리한다(‘연계 가능’이 아니라 ‘연계 방식’).
    

#### 1.2.2.4. 사회적 측면

(작성 지침)

- 사회 문제 해결 기여도 등
    

[(작성지침)]

- 사회적 효과는 “안전/숙련전수/청년인력 유입/지역 제조경쟁력” 중 최소 2개를 택한다.
    
- 안전은 “가상 실습으로 위험 작업을 대체”처럼 **행동 변화**를 중심으로 서술한다.
    
- 숙련전수는 “디지털 트윈 기반 표준 작업 시나리오”로 정의하고, 교육기관(폴리텍)과의 역할을 연결한다.
    
- 마지막 문장은 “현장 생산성 + 교육 접근성 + 안전성”이 함께 개선된다는 **삼중효과** 구조로 정리한다.
### 1.2.1. 과제의 중요성

[(작성지침)]

- 5개 세부항목을 아래 구조로 작성한다(각 항목 4~6줄): **현장문제(현재) → 손실/리스크 → 본 과제 해결요소 → 정량목표로 연결**
    
- 5개 항목 추천(예):
    
    1. 실시간성 부족(지연/누락) 문제 → “실시간 처리/동기화 목표”로 연결
        
    2. 2D 수치만으로는 상황판단 어려움 → 3D Twin 시각화 필요
        
    3. 장비 가동 중 교육·훈련 한계 → 안전/교육 가치
        
    4. 기종/프로토콜 종속 → 확장 가능한 연동 구조 필요
        
    5. 실증 이후 사업화 연결 필요 → **사업화율 130% 이상** 목표를 “왜 필요한가”의 결론으로 사용
        
- 고용 관련 공통목표(순고용증가 2명·신규채용 2명)는 “지역 디지털 제조 인력 기반 강화” 근거로 1~2문장 포함한다.

### 1.2.2. 과제의 파급효과

#### 1.2.2.1. 기술적 측면

[(작성지침)]

- “기술 파급”을 최소 4개로 쪼갠다:
    - (1) **Controller 데이터 정규화/데이터사전**(태그·단위·주기)
    - (2) **실시간 전송/동기화**(지연·누락·재연결)
    - (3) **Unity 상태머신 기반 동작 재현**(Run/Idle/Alarm, RPM/Feed/Load 매핑)
    - (4) **검증 체계(시험성적서)**(측정방법·산식·증빙)
        
- 마지막에 “지식재산권 1건 확보”를 기술적 성과의 귀결로 연결(미들웨어/매핑/복구 로직 등).
    
    

#### 1.2.2.2. 경제적·산업적 측면

[(작성지침)]

- 경제효과를 3갈래로 작성:
    
    1. **사업화 매출 창출**: 사업화율 130% 산식까지 계획에 명시(매출/지원금)
        
        2026년 테스트베드 경남 실증 지원사업 수행계획서(제출…
        
    2. **비용 절감**: (예) 장애 대응시간 단축, 시운전·교육비 절감(지표 정의 포함)
        
    3. **고용 창출**: 신규채용 2명 + 순고용증가 2명 달성 계획(채용월·직무·성과물)
        
        

#### 1.2.2.3. 디지털 혁신거점(디지털 산업) 측면

[(작성지침)]

- “경남 주력산업(제조/자동차/조선/우주항공)” 각각에 대해 **MCT 활용 공정 1개 + Twin 적용효과 1개**를 반드시 연결해 쓴다.
    
- 공통목표 중 **경남 디지털위크 참가 1건**을 ‘확산/홍보/수요발굴’ 실행계획으로 구체화(부스/전시물/데모 시나리오).
    
    2026년 테스트베드 경남 실증 지원사업 수행계획서(제출…
    
- “공동수행기관 협업 네트워크 1건”을 혁신거점 역할로 포지셔닝(산학연 공동수행, 네트워킹 행사, 후속 도입처 발굴).
    
    

#### 1.2.2.4. 사회적 측면

[(작성지침)]

- 사회적 효과는 최소 2개를 선택해 “증빙 가능한 활동”과 연결한다:
    
    - 안전/교육: 폴리텍 기반 실습 시나리오, 안전사고 예방
        
    - 숙련전수/청년 유입: 신규채용 2명(직무·역량)과 연계
        
- “컨설팅 1건”을 사회적 측면(기업 역량 강화)에도 연결 가능(운영/마케팅 분야 컨설팅 결과보고서 증빙).
