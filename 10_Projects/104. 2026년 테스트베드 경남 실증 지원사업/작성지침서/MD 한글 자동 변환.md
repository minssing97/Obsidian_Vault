한글에서 스타일 지정을 하는 방법은?

---

아래아한글에서 스타일을 지정하고 관리하는 방법은 크게 \*\*스타일 메뉴(F6)\*\*를 통한 설정과 \*\*개요 번호 모양(Ctrl+K, O)\*\*을 통한 연동 방식으로 나뉩니다. 사업계획서와 같이 구조가 복잡한 문서일수록 스타일 기반의 관리가 필수적입니다.

### 1\. 스타일 대화 상자 활용 (단축키: F6)

가장 표준적인 방법으로, 문서 전체의 일관성을 유지하기 위해 사용합니다.

- **스타일 수정 및 추가:**`F6` 을 눌러 스타일 창을 연 뒤, 수정하고 싶은 스타일(예: 개요 1)을 선택하고 하단의 \*\*연필 아이콘(스타일 편집)\*\*을 클릭합니다.
- **서식 지정:** 편집 창에서 \*\*글자 모양(L)\*\*과 \*\*문단 모양(T)\*\*을 각각 설정합니다. 예를 들어 '글자 모양'에서 폰트 종류와 크기(예: 24pt 굴림)를 지정하고, '문단 모양'에서 줄 간격이나 들여쓰기를 설정할 수 있습니다.
- **적용:** 설정을 마친 후 '설정' 버튼을 누르면 커서가 위치한 문단에 해당 스타일이 즉시 적용됩니다.

### 2\. 개요 번호와 스타일 연동 (단축키: Ctrl+K, O)

사업계획서의 번호 체계(Ⅰ, 1, 가 등)를 스타일과 연결하여 자동화하는 방법입니다.

- **번호 체계 설정:****\[서식\] - \[개요 번호 모양\]** 메뉴로 들어가 원하는 번호 체계를 선택하거나 **사용자 정의** 를 클릭합니다.
- **수준별 정의:** 각 수준(1수준~7수준)을 클릭하여 번호 모양(로마자, 아라비아 숫자 등)과 번호 뒤의 마침표/괄호 형식을 지정합니다.
- **스타일 연결:** 사용자 정의 창 내에서 각 수준에 해당하는 스타일을 연결해두면, 나중에 스타일만 적용해도 번호가 자동으로 매겨집니다.

### 3\. 실무 효율을 높이는 단축키 및 팁

- **빠른 스타일 적용:** 내용 입력 후 `Ctrl+Alt+1` ~ `9` 를 누르면 해당 숫자에 지정된 스타일(기본적으로 개요 1~9)이 즉시 적용됩니다.
- **수준 변경:** 이미 개요 스타일이 적용된 문단에서 `Ctrl` + `숫자패드 +` (수준 하강) 또는 `Ctrl` + `숫자패드 -` (수준 상승)를 눌러 단계를 빠르게 바꿀 수 있습니다.
- **서식 초기화:** 복사해서 붙여넣은 글의 번호가 꼬이거나 서식이 섞였다면, 해당 영역을 선택하고 `Ctrl+Space` 를 눌러 글자 모양을 초기화하거나 `Alt+Shift+C` 로 서식을 복사/적용하는 것이 확실합니다.

사업계획서 양식을 직접 만들 때는 **누름틀(Field)** 기능을 함께 활용하면, AI가 생성한 텍스트를 특정 스타일 위치에 정교하게 삽입하기 더욱 유리해집니다.

md 형식으로 작성한 원본을 한글로 옮기면 자동으로 스타일이 지정되게 하는 방법은?

---

마크다운(Markdown)으로 작성된 원본 데이터를 한글(HWP/HWPX)로 옮기면서 스타일을 자동으로 지정하는 가장 효과적인 방법은 **`pypandoc-hwpx` 라이브러리** 를 활용하거나, **`pyhwpx` 를 이용한 파이썬 스크립트** 를 작성하는 것입니다.

### 1\. pypandoc-hwpx를 이용한 자동 변환 (가장 추천)

이 도구는 마크다운의 계층 구조(H1~H6)를 한글의 개요 번호 체계(개요 1~6)로 직접 매핑해 줍니다. 특히 기존에 가지고 계신 사업계획서 양식 파일(`blank.hwpx`)의 스타일 정보(글꼴, 크기, 자간, 용지 설정 등)를 그대로 복제하여 새 문서에 적용할 수 있는 기능이 핵심입니다.

- **동작 원리:** 마크다운을 내부적으로 추상 구문 트리(AST)로 변환한 뒤, 참조용 파일(`--reference-doc`)에 정의된 스타일을 입혀 HWPX를 생성합니다.
- **명령어 예시:**
	Bash
	```
	pypandoc-hwpx input.md --reference-doc=사업계획서_양식.hwpx -o output.hwpx
	```
- **장점:** 이미지 처리 및 셀 병합이 포함된 복잡한 표도 지원하며, 한글 앱을 켜지 않고도 독립적으로 실행 가능합니다.

### 2\. pyhwpx를 이용한 스타일 매핑 스크립트

이미 한글 프로그램이 실행 중인 상태에서 파이썬을 통해 실시간으로 스타일을 지정하며 내용을 채우는 방식입니다. `hwp.set_style("스타일이름")` 메서드를 사용하면 특정 문단에 즉시 스타일을 입힐 수 있습니다.

- **자동화 로직:**
	1. 마크다운 파일의 각 행을 읽습니다.
	2. `#` 으로 시작하면 `hwp.set_style("개요 1")`, `##` 이면 `hwp.set_style("개요 2")` 를 실행하도록 조건문을 구성합니다.
	3. 일반 텍스트는 `hwp.insert_text()` 로 삽입 후 "바탕글" 스타일을 적용합니다.
- **장점:** 세밀한 조건부 스타일링이 가능하며, 특정 단어만 글자 색을 바꾸는 등의 정밀 제어가 가능합니다.

### 3\. AI에게 자동화 스크립트 작성을 요청하는 프롬프트

AI가 내용을 작성해 줄 때, 나중에 파이썬 스크립트로 처리하기 쉽도록 \*\*태그(Marker)\*\*를 붙여 달라고 요청하는 전략입니다.

**\[프롬프트 예시\]**

> "사업계획서 내용을 작성하되, 나중에 파이썬 스크립트가 인식할 수 있도록 각 문단 앞에 형식을 지정해 줘.
> 
> - 1단계 제목은 `[H1] 내용`
> - 2단계 제목은 `[H2] 내용`
> - 표 데이터는 ` tsv형식내용` 형식으로 작성해 줘. 이 텍스트를 받아서 `[H1]` 은 한글의 '개요 1' 스타일로, `[H2]` 는 '개요 2' 스타일로 매핑하여 한글 문서(.hwpx)를 자동 생성하는 파이썬(pyhwpx 활용) 코드를 작성해 줘."

### 4\. 수동 방식의 한계와 대안

단순히 마크다운을 HTML로 변환한 뒤 확장자만 `.hwp` 로 바꾸어 여는 방법도 있지만, 이 경우 한글 고유의 '개요 번호 체계'나 '스타일 목록'과 완벽히 연동되지 않아 추가 수정이 많이 필요합니다. 따라서 가급적 `pypandoc-hwpx` 와 같이 **참조 문서(Reference Doc) 기반의 변환** 을 사용하시는 것이 편집 시간을 가장 많이 줄여줍니다.

`pypandoc-hwpx` 는 문서 변환 도구인 Pandoc을 기반으로 하여, 마크다운(MD)이나 HTML, DOCX 파일을 아래아한글 표준 포맷인 **HWPX로 변환해 주는 파이썬 라이브러리** 입니다. 이 도구의 가장 큰 특징은 별도의 한글 프로그램 설치 없이도 작동하며, **기존 양식 파일의 스타일을 그대로 복제** 할 수 있다는 점입니다.

### 1\. 핵심 작동 원리

이 라이브러리는 Pandoc의 '참조 문서(Reference Document)' 개념을 활용합니다.

1. **AST(추상 구문 트리) 변환:** 입력받은 마크다운을 Pandoc이 내부적인 표준 구조(AST)로 먼저 해석합니다.
2. **스타일 지문 복제:** 사용자가 입력한 양식 파일(`--reference-doc`)에서 글꼴 모양(폰트, 크기, 속성), 문단 모양(줄 간격, 여백), 용지 설정 정보를 추출합니다.
3. **HWPX 생성:** 추출한 스타일 정보를 AI가 작성한 마크다운의 계층 구조에 입혀 최종적인 HWPX 파일을 생성합니다.

### 2\. 주요 장점 및 기능

- **스타일 자동 매핑:** 마크다운의 제목 태그(`#`, `##`, `###`)를 한글의 `개요 1`, `개요 2`, `개요 3` 스타일로 자동으로 연결합니다.
- **복잡한 표 지원:** 마크다운이나 HTML에서 작성된 복잡한 표 구조(셀 병합 포함)를 한글 표로 정확하게 구현합니다.
- **이미지 포함:** 마크다운 내에 포함된 이미지 경로를 추적하여 HWPX 문서 내부에 바이너리 형태로 삽입합니다.
- **독립적 실행:** 한글 소프트웨어가 설치되지 않은 리눅스 서버나 클라우드 환경에서도 문서 생성이 가능합니다.

### 3\. 설치 및 사용 방법

**설치 단계:**시스템에 Pandoc이 먼저 설치되어 있어야 하며, 이후 파이썬 환경에서 라이브러리를 설치합니다.

Bash

```
pip install pypandoc-hwpx
```

**변환 명령어:**터미널(CLI)에서 아래 명령어를 실행하여 AI가 쓴 마크다운 파일(`input.md`)에 사용자의 사업계획서 양식(`template.hwpx`) 스타일을 입힐 수 있습니다.

Bash

```
pypandoc-hwpx input.md --reference-doc=template.hwpx -o output.hwpx
```

### 4\. 실무 활용 팁

- **양식 파일(template.hwpx) 준비:** 양식 파일 내에 '개요 1'부터 '개요 6'까지의 스타일이 미리 정의되어 있어야 마크다운의 `#` 단계가 해당 스타일로 정확히 변환됩니다.
- **표 스타일 유지:** 양식 파일에 정의된 표의 기본 글자 모양과 문단 모양이 결과물에도 그대로 반영되므로, 기준이 되는 양식 파일의 표 서식을 미리 정리해 두는 것이 좋습니다.
- **한계점:** Pandoc의 표준 구조(AST)가 표현하지 못하는 아주 세밀한 한글 전용 기능(장평, 자간의 미세 조정 등)은 변환 과정에서 생략될 수 있으므로 최종 검토가 필요합니다.

파이썬에서 현재 어느 폴더에서 작업 중인지(현재 작업 디렉토리, Current Working Directory) 확인하는 방법은 크게 두 가지가 있습니다.

1. **os 모듈 활용**: 가장 전통적인 방법으로 `os.getcwd()` 함수를 호출합니다.
	Python
	```
	import os
	print(os.getcwd())
	```
2. **pathlib 모듈 활용**: 최신 파이썬 버전에서 권장되는 방식으로, 객체 지향적인 `Path.cwd()` 메서드를 사용합니다.
	Python
	```
	from pathlib import Path
	print(Path.cwd())
	```

`pyhwpx` 를 사용하여 문서를 불러올 때 (`hwp.open("FileName.hwp")`), 파일의 전체 경로를 입력하지 않으면 기본적으로 위에서 확인한 현재 작업 디렉토리에 있는 파일을 찾아서 엽니다. 만약 다른 폴더에 있는 파일을 열고 싶다면 `r"C:\path\to\file.hwpx"` 와 같이 전체 경로(절대 경로)를 입력해야 합니다.

터미널이나 명령 프롬프트(CMD)에서 현재 작업 디렉토리를 확인하는 방법은 운영체제에 따라 다릅니다.

1. **Windows (명령 프롬프트/CMD):**`cd` 명령어를 입력하고 엔터를 누르면 현재 경로가 표시됩니다.
2. **Windows (PowerShell):**`pwd` 명령어를 입력합니다.
3. **macOS 및 Linux:**`pwd` (Print Working Directory) 명령어를 입력합니다.

터미널에서 `cd [디렉토리명]` 명령어를 사용하여 특정 폴더로 이동하면, 해당 위치가 파이썬 스크립트가 실행되는 기본 작업 디렉토리가 됩니다. 파이썬에서 파일을 불러올 때 전체 경로를 적지 않으면 이 터미널 상의 현재 위치를 기준으로 파일을 찾게 됩니다.

이미지에 표시된 `NameError: name 'pypandoc' is not defined` 에러는 **'pypandoc-hwpx'를 파이썬 코드(변수나 함수)로 인식** 했기 때문에 발생한 것입니다. `pypandoc-hwpx` 는 파이썬 내부 명령어가 아니라 **터미널(CLI)에서 실행하는 외부 명령어** 입니다.

### 2\. 라이브러리 설치 확인

명령어를 실행하기 전에 해당 라이브러리가 설치되어 있는지 확인해야 합니다. 터미널이나 새 코드 셀에서 아래 명령을 실행하세요.

- **설치 명령어:**
	Bash

`pypandoc-hwpx` 는 내부적으로 **Pandoc** 이라는 프로그램을 사용합니다. 라이브러리를 설치했더라도 시스템에 Pandoc이 깔려 있지 않으면 실행 시 에러가 날 수 있습니다.

- **설치 여부 확인:** 터미널에 `pandoc --version` 을 입력해 보세요.

해당 에러(`RuntimeError: source_file is not a valid path`)는 **`input.md` 파일이 현재 작업 디렉토리에 존재하지 않거나, 경로가 잘못되었을 때** 발생합니다. 프로그램이 변환할 대상 파일을 찾지 못해 발생하는 전형적인 경로 오류입니다.

다음 단계에 따라 조치해 보시기 바랍니다.

### 1\. 파일 존재 여부 확인 (가장 빈번한 원인)

현재 주피터 노트북이나 터미널이 실행 중인 폴더에 실제로 `input.md` 파일이 있는지 확인해야 합니다. 아래 명령어를 코드 셀에 입력하여 파일 목록을 출력해 보세요.

- **Windows (CMD/Anaconda):**`!dir`
- **Mac/Linux:**`!ls`

목록에 `input.md` 가 없다면, 메모장이나 VS Code를 통해 해당 폴더에 파일을 생성하거나 해당 폴더로 파일을 옮겨야 합니다.

### 2\. 절대 경로 사용

파일이 다른 폴더에 있다면 파일 이름만 적지 말고 \*\*전체 경로(Absolute Path)\*\*를 입력하는 것이 가장 확실합니다.

Bash

```
# 예시: 파일이 C 드라이브의 작업 폴더에 있는 경우
!pypandoc-hwpx "C:/작업폴더/input.md" --reference-doc="C:/작업폴더/template.hwpx" -o "C:/작업폴더/output.hwpx"
```

*주의: 윈도우 경로를 복사해 붙여넣을 때 백슬래시(`\`)가 포함된다면 따옴표로 경로 전체를 감싸주는 것이 좋습니다.*

### 3\. 테스트용 파일 즉석 생성 (확인용)

만약 파일 생성법을 모르겠다면, 아래 코드를 실행하여 해당 위치에 `input.md` 파일을 강제로 만든 후 다시 변환 명령을 실행해 보세요.

Python

```
# 텍스트 파일(input.md) 생성 코드
content = """# 테스트 제목
본문 내용입니다.
"""
with open("input.md", "w", encoding="utf-8") as f:
    f.write(content)

print("input.md 파일이 생성되었습니다. 이제 변환 명령을 다시 실행하세요.")
```

### 4\. template.hwpx 파일 확인

`input.md` 문제를 해결하더라도, 참조용 양식 파일인 `template.hwpx` 가 없으면 동일한 에러가 발생할 수 있습니다. 이전에 사용하시던 사업계획서 양식 파일의 이름을 `template.hwpx` 로 변경하여 같은 폴더에 넣어두었는지 다시 한번 확인하시기 바랍니다.

### 요약

이 에러는 \*\*"명령어에 적힌 파일 이름이 실제 폴더에 있는 파일 이름과 다르거나 파일이 없다"\*\*는 뜻입니다. 파일명 오타를 확인하거나 파일의 전체 경로를 지정하면 해결됩니다.

이 에러 메시지(`OSError: No pandoc was found`)는 **`pypandoc-hwpx` 가 내부적으로 사용하는 문서 변환 도구인 'Pandoc'이 시스템에 설치되어 있지 않거나, 경로(PATH)가 설정되지 않아** 발생한 것입니다.

아나콘다(Anaconda) 환경을 사용 중이신 것으로 보이므로, 아래의 방법 중 하나를 선택하여 조치하시기 바랍니다.

### 방법 1: 파이썬 코드로 자동 설치 (가장 간편)

주피터 노트북이나 파이썬 셸에서 아래 코드를 실행하면 `pandoc` 프로그램을 현재 환경에 자동으로 다운로드하고 설치합니다.

Python

```
import pypandoc
pypandoc.pandoc_download.download_pandoc()
```

*설치가 완료된 후, 기존에 실행하려던 `!pypandoc-hwpx...` 명령어를 다시 실행해 보세요.*

### 방법 2: 아나콘다(Conda) 명령어로 설치

아나콘다 프롬프트나 터미널에서 아래 명령어를 입력하여 `pandoc` 패키지를 설치할 수 있습니다.

Bash

```
conda install -c conda-forge pandoc
```

### 방법 3: 수동 설치 및 경로 설정

위 방법들로 해결되지 않을 경우, Pandoc 공식 웹사이트에서 직접 설치 파일을 내려받아야 합니다.

1. **공식 사이트 접속:**[Pandoc 설치 페이지](https://pandoc.org/installing.html) 에서 Windows용 `.msi` 설치 파일을 다운로드합니다.
2. **설치 진행:** 설치 과정에서 **"Add Pandoc to the system PATH"** 옵션이 있다면 반드시 체크합니다.
3. **환경 변수 확인:** 설치 후에도 같은 에러가 난다면, \[내 PC 속성\] -> \[고급 시스템 설정\] -> \[환경 변수\]의 `Path` 항목에 Pandoc 설치 경로(예: `C:\Program Files\Pandoc`)가 추가되어 있는지 확인합니다.

### 추가 참고: 파일명 오타 주의

에러 메시지에 적힌 명령어를 보면 `input.md.md` 라고 되어 있습니다. 만약 실제 파일 이름이 `input.md` 라면 확장자가 중복 입력되어 나중에 다른 에러(`source_file is not a valid path`)가 발생할 수 있으니, 실행 시 파일명을 다시 한번 확인하시기 바랍니다.
