옵시디언의 기본 Dataview(DQL)는 메타데이터나 리스트(`-`), 할 일(`- [ ]`) 단위로 데이터를 처리하도록 설계되어 있어, `---`(가로 줄)로 구분된 일반 텍스트 단락을 직접 인식하여 추출하는 기능은 제공하지 않습니다.

하지만 **DataviewJS**를 활용하면 파일의 원문 전체를 읽어온 뒤 가로 줄을 기준으로 내용을 분리하고, 특정 태그가 포함된 섹션만 골라내어 화면에 뿌려줄 수 있습니다. 현재 작성하신 기출문제 노트의 구조에 딱 맞는 쿼리 예시입니다.

---

### **DataviewJS 쿼리 명령어**

아래 코드를 복습용 대시보드 노트에 입력하세요. `targetTag` 변수 부분에 원하는 태그(예: `#틀린문제` 또는 `#산업안전기사/암기`)를 넣으시면 됩니다.

JavaScript

```
const targetTag = "#산업안전기사/암기"; // 추출하고 싶은 태그 입력

// 1. 해당 태그가 포함된 모든 노트를 불러옵니다.
const pages = dv.pages("#산업안전기사"); 

for (const page of pages) {
    // 2. 파일의 전체 원문 내용을 읽어옵니다.
    const file = app.vault.getAbstractFileByPath(page.file.path);
    const content = await app.vault.read(file);
    
    // 3. '---'(가로 줄)을 기준으로 전체 내용을 나눕니다.
    const sections = content.split('---');
    
    // 4. 나뉜 섹션들 중 특정 태그를 포함하는 섹션만 필터링합니다.
    const matches = sections.filter(s => s.includes(targetTag));
    
    if (matches.length > 0) {
        // 파일 이름을 제목으로 표시 (선택 사항)
        dv.header(2, page.file.name);
        
        matches.forEach(m => {
            // 5. 조건에 맞는 섹션의 내용을 출력합니다.
            dv.paragraph(m.trim());
            // 시각적 구분을 위한 구분선 추가
            dv.el("hr", ""); 
        });
    }
}
```

---

### **작동 원리 설명**

1. **원문 읽기 (`app.vault.read`)**: Dataview가 미리 인덱싱한 메타데이터가 아니라, 파일에 적힌 글자 그대로를 가져옵니다.
    
2. **섹션 분리 (`split('---')`)**: 노트 곳곳에 배치된 가로 줄을 기준으로 내용을 '토막' 냅니다.
    
3. **태그 검사 (`filter`)**: 각 토막 안에 사용자가 지정한 태그(예: `#틀린문제`)가 들어있는지 하나씩 확인합니다.
    
4. **출력 (`dv.paragraph`)**: 조건에 맞는 토막들만 화면에 리스트업합니다.
    

이 방법을 사용하면 노트의 형식을 리스트(`-`)로 일일이 바꾸지 않아도, 가로 줄로 구분된 문제와 해설 전체를 통째로 불러와서 복습할 수 있습니다.

**추가 팁**: 만약 특정 과목의 문제들만 보고 싶다면 `const pages = dv.pages("#산업안전기사")` 부분을 더 구체적인 태그나 폴더 경로로 수정하여 범위를 좁힐 수 있습니다.

옵시디언(Obsidian)에서 앞서 안내해 드린 **DataviewJS** 코드를 실행하여 기출문제를 효율적으로 복습하는 방법은 다음과 같습니다.

### 1. Dataview 설정 확인

DataviewJS는 보안상의 이유로 기본적으로 비활성화되어 있을 수 있습니다. 먼저 설정을 확인해야 합니다.

1. 옵시디언 **설정(Settings)**으로 들어갑니다.
    
2. **커뮤니티 플러그인(Community plugins)** 메뉴에서 **Dataview**를 선택합니다.
    
3. 설정 항목 중 **Enable JavaScript Queries** 토글을 **ON**으로 켭니다.
    

### 2. 코드 블록 작성

일반적인 Dataview 쿼리와 달리, JavaScript 코드는 전용 코드 블록 안에 작성해야 합니다.

노트의 원하는 위치에 아래와 같이 입력하세요.

Markdown

````
```dataviewjs
const targetTag = "#산업안전기사/암기"; 

const pages = dv.pages("#산업안전기사"); 

for (const page of pages) {
    const file = app.vault.getAbstractFileByPath(page.file.path);
    const content = await app.vault.read(file);
    const sections = content.split('---');
    const matches = sections.filter(s => s.includes(targetTag));
    
    if (matches.length > 0) {
        dv.header(2, page.file.name);
        matches.forEach(m => {
            dv.paragraph(m.trim());
            dv.el("hr", ""); 
        });
    }
}
```
````

### 3. 결과 확인

- **라이브 프리뷰(Live Preview)** 모드에서는 커서를 코드 블록 바깥으로 옮기면 즉시 실행 결과가 나타납니다.
    
- **읽기 모드(Reading Mode)**에서는 항상 실행된 결과(특정 태그가 포함된 문제들)가 화면에 출력됩니다.

만약 옵시디언 외부에서 일반적인 JavaScript를 실행해보고 싶다면 아래 방법을 사용할 수 있습니다.
