## 1. 내부 링크의 개념과 중요성

우리의 뇌가 정보를 뉴런의 연결망으로 저장하듯, 노트도 서로 연결될 때 그 가치가 극대화됩니다.
개별 정보 조각들을 연결하면 의미 있는 '지식의 네트워크'가 만들어집니다.
옵시디언은 바로 이 '연결'을 핵심 철학으로 삼습니다. 
노트를 연결하는 과정은 단순히 정보를 정리하는 것을 넘어, 생각의 경계를 허물고 새로운 아이디어를 발견하도록 돕습니다. 
흩어져 있던 노트들이 연결되면서 예상치 못한 통찰력을 얻게 되는 것이 옵시디언의 진정한 힘입니다.

---
## 2. 가장 기본적인 내부 링크 만들기

옵시디언에서 내부 링크를 만드는 방법은 매우 간단합니다. 
대괄호 두 개(`[[ ]]`)로 연결하고 싶은 노트의 제목을 감싸기만 하면 됩니다.
`[[` 를 입력하면 내 보관소에 있는 노트 목록이 나타나 원하는 노트를 쉽게 선택할 수 있습니다.


**예시 : 아래와 같이 코딩하면** 
 ```markdown
 오늘 [[04_마크다운 문법 1 제목, 본문, 리스트 작성법 (위계 잡기)]] 을 복습했다.
 ```
 
 **아래와 같이 보입니다.**
 
  오늘 [[04_ 마크다운 문법 1 -제목, 본문, 리스트 작성법 (위계 잡기)]] 을 복습했다. 

아직 존재하지 않는 노트의 제목으로 링크를 만들어도 괜찮습니다. 
링크를 클릭(Ctrl/Cmd + 클릭)하면 해당 제목으로 새 노트가 즉시 생성됩니다. 
이를 통해 생각의 흐름을 끊지 않고 자유롭게 노트를 확장해 나갈 수 있습니다.

---
## 3. 링크 텍스트 변경하기 (파이프 트릭)

노트의 원제 대신 다른 이름으로 링크를 표시하고 싶을 때가 있습니다. 이때 파이프 기호(`|`)를 사용합니다. 
`[[파일이름|` 뒤에 원하는 텍스트를 입력하면, 링크는 '파일이름'으로 연결되지만 화면에는 '화면에 표시될 이름'으로 보이게 됩니다.
이를 통해 문장의 흐름을 해치지 않고 자연스러운 링크를 만들 수 있습니다.

 **아래와 같이 코딩하면**
 ```markdown
 오늘 배운 [[04_ 마크다운 문법 1 -제목, 본문, 리스트 작성법 (위계 잡기)|제4강]] 은 매우 유용했다.
 ```
 
 **아래와 같이 보입니다:**
 
 오늘 배운 [[04_ 마크다운 문법 1 -제목, 본문, 리스트 작성법 (위계 잡기)|제4강]] 은 매우 유용했다.

## 4. 특정 위치로 정밀하게 링크하기

노트 전체가 아닌, 노트 안의 특정 제목이나 문단으로 직접 연결하여 정보의 정확도를 높일 수 있습니다.

### 제목으로 링크하기
`[[파일이름#제목]]` 형식으로 링크합니다. `[[파일이름#`까지 입력하면 해당 파일의 제목 목록이 자동으로 나타나 쉽게 선택할 수 있습니다. 예를 들어 다음과 같이 링크하면, '마크다운 문법 1' 파일 내 '3. 목록 (Lists)' 제목으로 바로 이동합니다.

```markdown
[[마크다운 문법 1#3. 목록 (Lists)]]을 참고하여 리스트를 만들었다.
```

 
### 문단(블록)으로 링크하기
특정 문단, 이미지, 목록 항목 등 개별 블록에 연결하려면 블록 ID를 사용합니다.

1.  **블록 ID 생성**: 연결하고 싶은 문단 끝에 공백을 두고 `^원하는ID`를 추가합니다. (예: `이 문장은 중요합니다. ^b123`)
2.  **블록 링크 생성**: `[[파일이름#^블록ID]]` 형식으로 링크를 만듭니다.

`[[파일이름#^`까지만 입력해도 해당 파일의 블록 목록이 표시되어, ID를 직접 만들지 않고도 쉽게 링크할 수 있습니다.

 ```markdown
 // '아이디어 노트.md' 파일의 내용:
 오늘 회의에서 나온 아이디어는 매우 혁신적이었다. ^idea-01
 
 // 현재 노트에서 링크하기:
 [[아이디어 노트#^idea-01|지난 회의의 혁신적인 아이디어]]를 발전시켜보자.
 ```

## 5. 연결의 힘 발견하기

내부 링크의 진정한 힘은 연결된 노드들을 되짚어보고 전체 구조를 조망할 때 나타납니다.

### 백링크 (Backlinks)
백링크는 '현재 노트를 링크하고 있는 다른 모든 노트'의 목록을 보여주는 기능입니다. 사이드바의 백링크 창을 통해 어떤 노트들이 현재 노트를 참조하고 있는지 한눈에 파악할 수 있어, 생각의 맥락을 추적하고 지식을 확장하는 데 매우 유용합니다.

### 연결되지 않은 언급 (Unlinked Mentions)
백링크 창에서는 아직 링크로 연결되지 않았지만 현재 노트의 제목과 동일한 텍스트를 포함한 노트들도 찾아줍니다. '연결(Link)' 버튼을 클릭 한 번으로 이들을 실제 내부 링크로 손쉽게 변환하여 지식 네트워크를 더욱 촘촘하게 만들 수 있습니다.

### 그래프 뷰 (Graph View)
그래프 뷰는 내 보관소의 모든 노트와 그 연결 관계를 시각적인 지도로 보여주는 기능입니다. 어떤 노트가 중심적인 역할을 하는지, 어떤 아이디어들이 서로 군집을 이루는지, 또는 어떤 노트가 고립되어 있는지 직관적으로 파악할 수 있어 새로운 통찰을 얻는 데 도움을 줍니다.

## 6. (심화) 에일리어스(Aliases) 활용하기
에일리어스는 노트의 공식 제목 외에 여러 개의 '별명'을 부여하여 검색과 연결을 더욱 유연하게 만드는 강력한 기능입니다. 예를 들어, 'Personal Knowledge Management'라는 노트에 'PKM', '개인 지식 관리' 등의 별명을 붙여두면 어떤 키워드로든 쉽게 해당 노트를 찾고 링크할 수 있습니다.

### 에일리어스 설정 방법
에일리어스는 노트 최상단의 **프론트매터(Frontmatter)** 영역에 설정합니다. 프론트매터는 파일의 시작 부분에 `---`로 감싸인 부분을 의미합니다.

**예시:**
```markdown
---
aliases:
  - 내부링크 활용
  - 옵시디언 링크
tags:
  - 경상대
  - 옵시디언강의
Theme:
  - obsidian
  - Link
---
```
위와 같이 설정하면, '06_연결의 마법 1...' 이라는 긴 원제 대신 '내부링크 활용'이나 '옵시디언 링크'라는 키워드로 이 노트를 쉽게 찾고 연결할 수 있습니다.

### 에일리어스 활용
-   **유연한 링크**: `[[`를 입력하고 에일리어스로 지정한 '내부링크 활용'을 타이핑해도 원본 노트를 찾아 링크할 수 있습니다. 문맥에 맞는 자연스러운 이름으로 노트를 연결할 때 매우 유용합니다.
-   **빠른 검색**: 빠른 전환(Ctrl/Cmd + O) 창에서 에일리어스를 검색하여 원하는 노트로 즉시 이동할 수 있습니다. 긴 파일명을 모두 기억할 필요가 없어집니다.
-   **다양한 키워드 관리**: 약어(AI), 원어(Obsidian), 동의어(지식 베이스) 등 다양한 형태의 키워드를 에일리어스로 등록해두면, 어떤 단어가 떠오르든 해당 정보에 막힘없이 접근할 수 있어 지식 네트워크의 밀도를 높여줍니다.
