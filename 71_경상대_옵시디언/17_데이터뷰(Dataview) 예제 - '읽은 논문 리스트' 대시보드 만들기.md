## 1. 논문 노트 관리를 위한 메타데이터 설계하기
-   '읽은 논문' 목표 설정 및 필요성 이해
    -   논문을 꾸준히 읽고 관리하는 것은 지식 체계를 구축하고, 연구 동향을 파악하며, 필요할 때 참고 자료를 빠르게 찾기 위해 중요합니다. 옵시디언과 데이터뷰를 활용하면 읽은 논문들을 단순히 쌓아두는 것을 넘어, 서로 연결하고 분류하여 자신만의 지식 데이터베이스로 만들 수 있습니다.

-   논문 관리에 필요한 메타데이터 항목 정의 (저자, 발행연도, 주제 등)
    -   메타데이터는 '데이터에 대한 데이터'로, 논문 노트의 핵심 정보를 담습니다. 어떤 메타데이터를 사용할지 미리 정의해야 체계적인 관리가 가능합니다. 예컨대 다음과 같은 설계가 가능하겠지요.
        -   `author`: 저자
        -   `year`: 발행 연도
        -   `journal`: 학술지 또는 컨퍼런스명
        -   `topic`: 논문의 주요 주제 (데이터뷰 그룹화에 활용)
        -   `rating`: 개인적인 중요도나 평점 (1~5)
        -   `status`: 읽기 상태 (예: `읽을 것`, `읽는 중`, `완독`)

-   YAML 프론트매터 형식으로 메타데이터 구조 설계하기
    -   YAML 프론트매터는 마크다운 파일 최상단에 `---`로 감싸서 작성하는 영역입니다. 데이터뷰는 이 정보를 활용하여 노트를 분석하고 시각화합니다. 위에서 정의한 항목들을 아래와 같이 YAML 형식으로 정리할 수 있습니다.
    
        ```yaml
        ---
        author: [Author Name1, Author Name2]
        year: 2023
        journal: "Journal of Knowledge Management"
        topic: [dataview, obsidian, pkm]
        rating: 5
        status: 완독
        ---
        ```

## 2. 템플릿으로 효율적인 논문 노트 만들기
-   옵시디언 템플릿 기능 활성화 및 설정
    -   옵시디언의 코어 플러그인 '템플릿(Templates)'을 사용하면 반복적인 노트 구조를 쉽게 만들 수 있습니다. 먼저 이 기능을 활성화해야 합니다.
    -   `설정` > `코어 플러그인`으로 이동하여 `템플릿`을 활성화합니다.
    -   `설정` > `템플릿`에서 `템플릿 폴더 위치`를 지정합니다. (예: `templates` 폴더)

-   메타데이터를 포함한 논문 노트 템플릿 작성하기
    -   지정한 템플릿 폴더에 1단계에서 설계한 YAML 프론트매터가 포함된 새 노트를 만듭니다. 이렇게 하면 모든 논문 노트가 일관된 구조를 갖게 됩니다.
    
        **`templates/논문 템플릿.md` 예시:**
        ```markdown
        ---
        author: []
        year: 
        journal: ""
        topic: []
        rating: 
        status: 읽을 것
        ---
        
        # {{title}}
        
        ## 초록 (Abstract)
        
        ## 핵심 내용
        
        ## 내 생각 및 비판
        
        ```

-   템플릿을 사용하여 새로운 논문 노트 생성 및 데이터 입력하기
    -   새로운 논문 노트를 만들 때마다 이 템플릿을 적용할 수 있습니다.
    -   새 노트를 생성한 후, `Cmd/Ctrl + P`를 눌러 커맨드 팔레트를 열고 '템플릿 삽입(Insert template)'을 실행하여 만들어 둔 템플릿을 선택합니다.
    -   템플릿이 적용되면 YAML 프론트매터의 빈칸과 각 섹션을 논문 내용에 맞게 채워넣습니다.

## 3. 데이터뷰(Dataview)로 '읽은 논문 리스트' 대시보드 만들기
-   기본 데이터뷰 쿼리 작성법 배우기 (`TABLE`)
    -   데이터뷰 쿼리는 ` ` `dataview` 코드 블록 안에 작성합니다. 가장 기본이 되는 `TABLE` 쿼리는 지정한 노트들의 정보를 표 형태로 보여줍니다.
    -   `TABLE author, year, rating` 처럼 YAML 프론트매터의 필드명을 나열하면 해당 정보가 표의 열이 됩니다.

-   `FROM`을 사용하여 특정 폴더/태그의 노트 가져오기
    -   `FROM` 구문은 데이터뷰가 검색할 범위를 지정합니다. 특정 폴더(`"논문"`), 특정 태그(`#paper`), 또는 두 가지를 조합하여 사용할 수 있습니다.
    -   `FROM`을 생략하면 볼트 전체에서 노트를 검색합니다. `WHERE`를 사용하면 더 상세한 조건(예: `status = "완독"`)으로 노트를 필터링할 수 있습니다. 다음과 같이 쿼리해 보세요.

```
	```dataview
		TABLE author, year, rating
		FROM "논문"
		WHERE status = "완독"
	```
```

-   `GROUP BY`를 사용하여 주제별/연도별로 논문 그룹화하기
    -   `GROUP BY`는 특정 필드를 기준으로 노트를 그룹화하는 강력한 기능입니다. 예를 들어, `GROUP BY topic`을 사용하면 `topic` 필드값이 같은 논문들이 하나의 그룹으로 묶입니다.
    -   그룹화된 데이터는 `rows`라는 객체를 통해 접근할 수 있습니다. 예를 들어, `rows.file.link`는 그룹에 속한 모든 노트의 링크 목록을 의미합니다.

-   완성된 대시보드 쿼리 작성 및 결과 확인하기
    -   지금까지 배운 `TABLE`, `FROM`, `WHERE`, `GROUP BY`를 조합하여 주제별로 완독한 논문을 정리하는 대시보드를 만들 수 있습니다.

        **'읽은 논문 리스트' 대시보드 쿼리 예시:**

```
	```dataview
        TABLE WITHOUT ID
        	topic AS "주제",
        	rows.file.link AS "논문 제목",
        	rows.author AS "저자",
        	rows.rating AS "평점"
        FROM "논문"
        WHERE status = "완독"
        GROUP BY topic
        SORT topic ASC     
	```
```


-   위 쿼리는 "논문" 폴더에서 `status`가 "완독"인 노트를 찾아 `topic`별로 그룹화합니다.
    -   `WITHOUT ID`는 파일명을 첫 번째 열에 표시하지 않도록 하는 옵션입니다.
    -   `AS`를 사용하여 표의 헤더(열 이름)를 원하는 텍스트로 변경할 수 있습니다.
    -   `SORT topic ASC`는 주제를 기준으로 오름차순 정렬합니다.
