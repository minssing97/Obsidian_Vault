## 1. 왜 메타데이터인가?: 정보에 질서를 부여하는 기술
- **흩어진 노트를 '정보 자산'으로 바꾸기** : 메타데이터는 '데이터에 대한 데이터'입니다. 노트에 저자, 생성일, 주제 등의 정보를 명시적으로 부여함으로써, 단순한 텍스트 덩어리를 검색하고, 분류하고, 연결할 수 있는 체계적인 '정보 자산'으로 만들어 줍니다.
- **옵시디언의 3가지 메타데이터: 태그(Tags), 속성(Properties), 인라인 필드(Inline Fields)**
    - **태그**: `#주제`처럼 간단하게 노트를 분류하는 꼬리표입니다.
    - **속성**: `author: Tolkien`처럼 노트 전체에 대한 구조적인 정보를 상단에 부여합니다.
    - **인라인 필드**: 문장 중간에 `Source:: The Hobbit`처럼 특정 데이터 값을 기록합니다.
- **어떤 메타데이터를 언제 사용해야 할까?**
    - **태그**: 빠르고 넓은 범위의 분류가 필요할 때 (예: `#아이디어`, `#독서`)
    - **속성**: 노트의 정체성을 정의하는 체계적인 정보가 필요할 때 (예: 책 노트의 저자, 출판년도, 평점)
    - **인라인 필드**: 글의 맥락 속에서 특정 데이터를 기록하고 싶을 때 (예: 인용문의 출처, 특정 날짜의 운동 기록) 

## 2. 기초 다지기: 태그(Tags)를 이용한 빠른 분류
- **태그의 기본 사용법: `#주제`, `#상태/진행중`**
    - 노트 본문 어디에서든 `#` 기호 뒤에 띄어쓰기 없이 단어를 입력하면 태그가 됩니다.
    - 슬래시(`/`)를 사용하면 `상태`라는 큰 그룹 아래 `진행중`이라는 하위 태그를 만드는 계층 구조를 만들 수 있습니다.
- **태그 창(Tag pane)을 활용한 즉각적인 필터링**
    - 코어 플러그인인 '태그 창'을 활성화하면 사이드바에 보관함의 모든 태그 목록이 나타납니다. 태그를 클릭하기만 하면 해당 태그가 포함된 모든 노트를 즉시 필터링해서 볼 수 있습니다.
- **태그의 장점과 한계: 언제 태그만으로 충분할까?**
    - **장점**: 입력이 매우 빠르고 간편하며, 즉각적인 분류와 필터링에 강력합니다.
    - **한계**: `저자: Tolkien`과 같은 '키-값' 형태의 구조적인 데이터를 저장할 수 없습니다. `#Tolkien`이라는 태그를 달 수는 있지만, 이것이 '저자'를 의미하는지는 알 수 없습니다.
    - 간단한 주제 분류나 상태 표시(예: `#읽을것`)만 필요하다면 태그만으로도 충분합니다.

## 3. 핵심 기능 마스터: 속성(Properties)으로 노트에 명찰 달기
- **속성(Properties)이란? (YAML Frontmatter)**
    - 속성은 노트의 최상단에 `---`로 감싸인 '프론트매터(Frontmatter)' 영역에 저장되는 `키: 값` 형태의 구조화된 데이터입니다. 노트 전체의 정체성을 정의하는 정보(예: 책의 저자, 문서의 상태)를 기록하는 데 사용됩니다.
- **UI를 이용한 손쉬운 속성 추가 (단축키: `Ctrl + ;`)**
    - 노트 제목 아래 '파일 속성 추가' 버튼을 누르거나, 단축키 `Ctrl + ;` (macOS: `Cmd + ;`)를 사용하면 깔끔한 UI를 통해 손쉽게 속성을 추가하고 관리할 수 있습니다.
- **텍스트, 숫자, 날짜, 체크박스, 리스트 등 다양한 속성 유형 활용하기**
    - 속성은 단순 텍스트 외에도 다양한 데이터 유형을 지원하여 정보를 더욱 체계적으로 관리할 수 있게 해줍니다. (예: `rating: 5`, `read: true`, `published: 1937-09-21`)
- **활용 사례: 독서 노트(저자, 평점, 출판년도), 회의록(참석자, 날짜), 프로젝트(상태, 마감일)**
    - **독서 노트**: `author`, `rating`, `published_year` 속성을 추가하여 나만의 독서 데이터베이스를 만들 수 있습니다.
    - **회의록**: `attendees`, `meeting_date` 속성으로 회의 정보를 체계적으로 관리합니다.
    - **프로젝트**: `status` (예: 진행중, 완료), `due_date` 속성으로 프로젝트 현황을 한눈에 파악할 수 있습니다.

## 4. 인라인 필드(Inline Fields)로 문맥 속에 데이터 심기
- **인라인 필드란?: `Key:: Value` 형식**
    - 인라인 필드는 노트 본문 아무 곳에나 `키:: 값` (콜론 두 개 사용) 형식으로 작성하는 메타데이터입니다. Dataview 플러그인에서 널리 사용되며, 이제 옵시디언 기본 검색에서도 인식됩니다.
- **속성(Properties)과의 차이점: 노트 전체 정보 vs. 노트 속 특정 정보**
    - **속성**: 노트 전체에 대한 정보를 다룹니다. (예: 이 '호빗' 노트의 저자는 톨킨이다.)
    - **인라인 필드**: 노트 내용의 특정 부분, 즉 문맥 속 정보를 다룹니다. (예: 이 '인용문'의 출처는 호빗이다.)
- **활용 사례: 문장 출처 표기(`Source:: BookName`), 습관 추적(`운동시간:: 30분`)**
    - 인용한 문장 바로 뒤에 `출처:: [[호빗]]`과 같이 적어 출처를 명확히 할 수 있습니다.
    - 데일리 노트에 그날의 활동을 기록하며 `운동시간:: 30분` 또는 `수면시간:: 7시간`과 같이 데이터를 기록하여 나중에 추적하고 분석할 수 있습니다.


## 5. 메타데이터, 날개를 달다: 검색과 자동화
- **태그, 속성, 인라인 필드를 활용한 정교한 검색 방법 복습**
    - 이제 메타데이터를 이용해 옵시디언의 기본 검색 기능을 더욱 강력하게 활용할 수 있습니다.
    - **태그 검색**: `tag:#독서`
    - **속성 검색**: `[author:Tolkien]` 또는 `[rating:>4]`
    - **인라인 필드 검색**: `[운동시간]` (해당 필드가 있는 모든 노트)
- **Dataview 플러그인: 메타데이터를 활용해 노트를 자동으로 목록화하기**
    - Dataview는 가장 강력한 커뮤니티 플러그인 중 하나로, 입력된 메타데이터를 기반으로 노트를 데이터베이스처럼 조회하고, 자동으로 목록이나 표를 만들어주는 도구입니다. 더 이상 수동으로 목록을 관리할 필요가 없어집니다.
- **나만의 대시보드 만들기: '진행 중인 프로젝트 목록', '최근 읽은 책 목록' 자동 생성하기**
    - Dataview 쿼리를 활용하면, 여러 정보를 한곳에 모아보는 '대시보드' 노트를 만들 수 있습니다. 이 대시보드는 관련 노트의 내용이 변경되면 자동으로 업데이트됩니다.
    - **예시: 진행 중인 프로젝트 목록 만들기**
        - `status` 속성이 "진행중"인 노트를 찾아 마감일과 함께 표로 보여줍니다.

```
	```dataview
	TABLE due_date as "마감일"
	FROM #프로젝트 
	WHERE status = "진행중"
	SORT due_date asc
	```
```
