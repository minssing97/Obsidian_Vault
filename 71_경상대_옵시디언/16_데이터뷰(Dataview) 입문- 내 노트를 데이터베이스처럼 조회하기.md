## 1. 메타데이터: 노트에 데이터 추가하기
- **데이터뷰란 무엇인가? (노트를 데이터베이스처럼 만드는 플러그인)**
  - 옵시디언 노트를 데이터베이스처럼 조회하고, 필터링하며, 목록을 만들어주는 강력한 커뮤니티 플러그인입니다.
  - 노트에 '데이터'를 부여하고, 이 데이터를 기반으로 동적인 목차나 대시보드를 생성할 수 있습니다.
- **메타데이터의 두 가지 종류와 작성법**
  - 메타데이터(Metadata)는 '데이터에 대한 데이터'로, 노트의 속성(예: 생성일, 태그, 상태)을 의미합니다. 데이터뷰는 이 메타데이터를 읽어 작동합니다.
  - **YAML 프론트매터(Frontmatter): 노트 최상단에 `---`로 감싸서 작성하는 형식**
    - 파일의 가장 첫 부분에 `---` 세 개로 시작하고,  `key: value` 형태로 작성합니다.
    - 다 작성한 다음에는 `---` 로 마무리하면 됩니다.
    - 노트 전체에 적용되는 정보를 구조적으로 관리하기 좋습니다.
    - 예시:
      ```yaml
      ---
      type: "Book"
      author: "Yuval Noah Harari"
      rating: 5
      read_date: 2023-10-27
      tags: [history, non-fiction]
      ---
      ```
  - **인라인 필드(Inline Field): 노트 본문 아무 곳에나 `key:: value` 형식으로 작성**
    - 문장 중간이나 원하는 위치에 `Key:: Value` (콜론 두 개) 형식으로 데이터를 추가할 수 있습니다.
    - 문맥 속에서 데이터를 추가하거나 간단한 정보를 빠르게 입력할 때 유용합니다.
    - 예시:
      - 프로젝트 상태: `상태:: 진행중`
      - 마감일 정보: 이 작업은 `(마감일:: 2023-11-15)` 까지 완료해야 합니다.
      - 키에 공백이 있는 경우: `[프로젝트 리더]:: 홍길동`

## 2. 기본 쿼리: 노트 목록 만들기 (LIST, TABLE)
- **데이터뷰 쿼리 블록 작성법**
  - 데이터뷰 쿼리는 `` ```dataview `` 로 시작하고 ` ``` ` 로 끝나는 코드 블록 안에 작성합니다.
  - 이 블록 안의 쿼리는 미리보기(Reading View) 모드에서 동적인 목록이나 표로 자동 변환됩니다.

- **`LIST`: 가장 간단한 파일 목록 조회**
  - 지정한 조건(`FROM`)에 맞는 노트들을 간단한 파일 목록 형태로 보여줍니다.
  - 예시: `#경상대` 태그를 가진 모든 노트 목록 조회

```
```dataview
LIST
FROM #경상대 
```

위와 같이 쿼리하면, 다음과 같이 리스트를 만들 수 있습니다.

```dataview
LIST
FROM #경상대 
```

- **`TABLE`: 특정 필드(메타데이터)를 표 형태로 조회**
  - `TABLE` 뒤에 보고 싶은 메타데이터 필드(key)들을 쉼표로 나열하면, 해당 필드들이 표의 열이 되어 나타납니다.
  - 첫 번째 열은 파일 이름(링크)이 자동으로 생성됩니다.
  - 예시: `#경상대` 태그를 가진 노트들의 `alises`, `theme` 필드를 표로 조회


![[Pasted image 20251209213423.png]]

위와 같이 쿼리하면, 다음과 같은 테이블을 만들 수 있습니다.
```dataview
TABLE aliases, theme
FROM #경상대                            
```

- **`FROM`: 데이터 조회 대상 지정하기**
  - 쿼리를 실행할 노트의 범위를 지정합니다.
  - **태그**: `FROM #태그이름` (예: `FROM #project`)
  - **폴더**: `FROM "폴더경로"` (예: `FROM "Meetings"`)
  - **여러 조건 조합**: `AND`, `OR`를 사용하여 조건을 조합할 수 있습니다.
    - 예시 1: `#task` **그리고** `#work` 태그를 모두 가진 노트 (`FROM #task AND #work`)
    - 예시 2: `"Projects"` 폴더에 있**거나** `#project` 태그를 가진 노트 (`FROM "Projects" OR #project`)

## 3. 쿼리 심화: 노트 필터링하고 정렬하기 (WHERE, SORT)
- **`WHERE`: 원하는 조건에 맞는 노트만 필터링하기**
  - `FROM`으로 가져온 노트 목록에서 특정 조건을 만족하는 노트만 선택하는 강력한 필터 기능입니다.
  - 다양한 비교 연산자( =,  !=, >, <, >=, <=)를 사용할 수 있습니다.

  - 예시 1: '상태'가 "완료"인 노트만 조회

![[IMG_0182.jpeg]]
 
  - 예시 2: '평점'이 4 이상인 책만 조회

![[IMG_0182 1.jpeg]]

- **`SORT`: 특정 필드를 기준으로 결과 정렬하기**
  - 조회된 결과를 특정 필드(메타데이터) 값을 기준으로 정렬합니다.
  - `ASC` (Ascending): 오름차순 (기본값, 생략 가능)
  - `DESC` (Descending): 내림차순
  - 예시: 읽은 책 목록을 '평점'이 높은 순서대로 정렬

![[IMG_0184.jpeg]]

  - `#project` 태그를 가지고 있고, `status` 필드가 "진행중"인 노트들을 찾아서, 각 프로젝트의 `담당자`와 `마감일`을 표로 만듭니다.
