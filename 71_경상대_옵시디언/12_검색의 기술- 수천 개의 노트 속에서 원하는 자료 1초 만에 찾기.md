## 1. 검색의 첫걸음: 기본 기능 마스터하기
- **기본 검색 연산자: 따옴표(""), 파일명(file:), 경로(path:)**
    - `"정확한 구문"`: 단어의 순서까지 일치하는 구문을 찾습니다. 예: `"옵시디언 검색"`
    - `file:파일이름`: 노트의 내용이 아닌 파일 제목을 기준으로 검색합니다. 예: `file:검색의 기술`
    - `path:폴더경로`: 특정 폴더 내의 노트로 검색 범위를 제한합니다. 예: `path:"강의자료/"`
- **검색 결과 저장 및 재활용: 검색 쿼리 북마크하기**
    - 자주 사용하는 복잡한 검색어는 '북마크' 기능을 통해 저장하고 클릭 한 번으로 다시 실행할 수 있습니다. (코어 플러그인 '북마크' 활성화 필요)
    - 검색 결과창 우측의 `...` 메뉴에서 '이 검색어 북마크하기'를 선택하여 저장합니다.

## 2. 연결을 통해 검색하기: 태그와 링크 활용법
- **태그(#)를 이용한 주제별 자료 분류 및 검색**
    - `tag:` 연산자를 사용하여 특정 태그가 포함된 노트를 쉽게 찾을 수 있습니다. 예: `tag:#프로젝트`
- **특정 노트를 참조하는 모든 노트 찾기 (별명 포함)**
    - 검색창에 `"[[파일이름"` 과 같이 링크의 시작 부분을 입력하면, 별명(`|`)이 사용된 링크까지 모두 찾을 수 있습니다. 링크의 시작 텍스트가 동일하기 때문입니다.
    - 예: `"[[옵시디언 검색"`으로 검색하면, `[[옵시디언 검색]]`과 `[[옵시디언 검색|검색 꿀팁]]`이 포함된 노트를 모두 찾을 수 있습니다.
- **연결되지 않은 고아 노트(Orphan notes) 발견하고 연결하기**
    - 고아 노트(Orphan notes)는 다른 어떤 노트와도 연결되지 않은 고립된 노트를 의미합니다.
    - 기본 검색만으로 찾기는 어렵지만, 그래프 뷰에서 시각적으로 확인하거나 Dataview 같은 플러그인을 활용하여 `length(file.inlinks) = 0` 조건으로 쉽게 찾을 수 있습니다.

## 3. 검색 쿼리 조합하기: 불리언(Boolean) 연산과 작업 관리
- **`AND`, `OR`, `-`(NOT)을 활용한 정교한 검색**
    - **AND**: 두 단어 사이에 공백을 두면 `AND` 연산으로 처리됩니다. 예: `옵시디언 검색` (옵시디언과 검색이 모두 포함된 노트)
    - **OR**: `OR`을 사용해 둘 중 하나라도 포함된 노트를 찾습니다. 예: `옵시디언 OR 노션`
    - **- (NOT)**: `-`를 단어 앞에 붙여 특정 단어를 제외합니다. 예: `옵시디언 -플러그인` (옵시디언은 포함, 플러그인은 제외)
- **`( )`를 이용한 검색 조건 그룹화**
    - 여러 조건을 조합할 때 괄호를 사용하면 연산의 우선순위를 지정할 수 있습니다.
    - 예: `(옵시디언 OR 노션) tag:#정리` (옵시디언 또는 노션이 포함되고, `#정리` 태그가 있는 노트)

## 4. 노트를 데이터베이스처럼: 속성(Properties) 검색
- **속성(Properties) 기능 소개 및 활용법**
    - 노트 상단에 `key: value` 형식으로 추가하는 메타데이터입니다. `status: 진행중`, `rating: 5` 와 같이 노트에 체계적인 정보를 부여할 수 있습니다.
- **`[key:value]` 문법으로 특정 속성값을 가진 노트 검색하기**
    - 대괄호를 이용해 특정 속성값을 가진 노트를 검색할 수 있습니다.
    - 예: `[status:완료]` (status 속성의 값이 '완료'인 모든 노트)
- **숫자, 날짜 범위 검색을 통한 데이터 필터링**
    - 부등호를 사용해 숫자나 날짜의 범위를 지정하여 검색할 수 있습니다.
    - 숫자 예: `[rating:>3]` (rating 속성값이 3보다 큰 노트)
    - 날짜 예: `[due:<2024-01-01]` (due 속성 날짜가 2024년 1월 1일 이전인 노트)

## 5. 검색 자동화와 전문가 팁: 임베디드 쿼리
- **`query` 코드 블록: 검색 결과를 노트에 실시간으로 임베딩하기**
    - `query` 코드 블록을 사용하면 검색 결과를 노트 안에 직접 삽입할 수 있습니다. 이 목록은 실시간으로 자동 업데이트되어, 노트가 추가되거나 내용이 변경될 때마다 항상 최신 상태를 반영합니다.
    - **사용법**: ` ```query ` 로 코드 블록을 시작하고, 내부에 검색창에서 사용하던 검색어를 그대로 입력합니다.
    - **예시**: ` ```query tag:#프로젝트 [status:진행중] ``` `
- **나만의 대시보드(Dashboard) 구축하기**
    - 대시보드는 여러 `query` 코드 블록을 조합하여 중요한 정보를 한곳에 모아보는 노트입니다. '오늘 할 일', '진행 중인 프로젝트', '새로 추가된 아이디어' 등의 섹션을 만들고 각각에 맞는 검색 쿼리를 임베딩하여 개인화된 작업 공간을 만들 수 있습니다.
    - 이를 통해 매번 정보를 찾아다닐 필요 없이 대시보드 노트 하나만으로 전체 상황을 빠르고 정확하게 파악할 수 있습니다.
